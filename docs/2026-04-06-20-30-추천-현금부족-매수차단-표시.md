# 추천 현금 부족 매수 차단 표시

## 프롬프트

> 추천시 내 캐시를 보고 추천해야 할것 같다
> 고고

## 문제

- 추천 파이프라인은 BUY 합의 종목에 `action_plan`을 붙이지만, `available_cash`가 현재가 1주보다 작아도 1주 매수안을 강제로 만들 수 있었다.
- 이 경우 추천 목록에는 매수 전략이 표시되어 실제 자금 사정과 맞지 않는 안내가 나올 수 있었다.

## 변경 사항

### `src/portfolio/sizer.py`

- `compute_buy_plan()`에서 `max_shares_by_cash < 1`이면 `buy_blocked`를 반환하도록 변경
- 차단 사유에 `가용 현금 < 1주 필요금액`을 금액과 함께 명시
- 추가로 `max_shares_by_weight < 1` 또는 손절 기준상 목표 수량이 0주인 경우도 `buy_blocked`로 반환하도록 보강
- 기존처럼 추천 목록 자체는 유지하고, 출력 레이어에서 이미 지원하던 `매수 차단` 문구를 그대로 사용

## 결과

- 현재 현금으로 1주도 살 수 없는 종목은 추천 목록에 남되, 허위 매수 전략 대신 `매수 차단`으로 표시된다.
- 검증 커맨드

```bash
uv run python -m py_compile src/portfolio/sizer.py
uv run python -c "from src.portfolio.sizer import compute_buy_plan; portfolio={'cash':400000,'positions':[]}; portfolio_check={'can_buy':True,'buy_block_reason':None,'total_assets':1000000,'available_cash':400000}; config={}; print(compute_buy_plan(550000,495000,'high',portfolio,portfolio_check,config))"
```

- 검증 결과: `{'type': 'buy_blocked', 'reason': '가용 현금 400,000원 < 1주 필요금액 550,000원'}`

## 참고

- `lsp_diagnostics`에는 이 파일의 기존 basedpyright 타입 경고/오류가 다수 남아 있다. 이번 수정으로 추가된 신규 구문 오류는 없고, 검증 커맨드도 정상 통과했다.
