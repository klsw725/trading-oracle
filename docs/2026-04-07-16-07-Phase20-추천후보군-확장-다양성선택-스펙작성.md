# Phase20 추천 후보군 확장 + 다양성 선택 스펙 작성

## 프롬프트

> 새로운 추천 스펙 문서를 작성하되, `docs/specs/v3/SPEC.md`와 기존 v3 PRD 예시 형식을 참고해서 만들어라. 또한 `US`가 있다면 `KR`이 있어야 하고, `ALL`은 `KOSPI + KOSDAQ`가 아니라 `KR + US` 의미가 되도록 설계하라.

## 변경 사항

### `docs/specs/recommend/SPEC.md`

- 추천 파이프라인 재설계를 위한 신규 스펙 폴더 생성
- 스펙 문서에 다음 핵심 개념 정리
  - `KR`, `US`, `ALL` 시장 semantics
  - 넓은 universe 확보 후 `top_n` 압축
  - 시장/섹터 다양성 선택 규칙
  - 기존 Phase 8 파이프라인과의 관계

### `docs/specs/recommend/prds/phase20-diversified-recommend.md`

- Phase 20 신규 PRD 문서 추가
- 추천 스크리닝 재설계 방향을 다음 기준으로 정리
  - 시장 옵션 의미 재정의 (`KR`, `US`, `ALL`)
  - 넓은 후보군 확보 후 최종 `top_n` 분석 대상으로 압축
  - 시장/섹터 편중을 줄이는 다양성 선택 로직
  - 점진적 제약 완화와 JSON 메타데이터 노출
- 기존 v3 PRD 스타일에 맞춰 문제 / 솔루션 / 마일스톤 / 데이터 흐름 / 설정 / 안전장치 / 비용 / 리스크 / 구현 순서 섹션으로 구성

## 결과

- 추천 파이프라인 재설계 방향이 별도 spec namespace 아래 문서화되어 이후 구현 시 기준점으로 사용할 수 있다.
- `KR`, `US`, `ALL` 시장 semantics를 대칭적으로 정리해 사용자 혼동을 줄인다.
- 스펙 제목은 기능 버전명 대신 더 일반적인 `Recommendation Pipeline` 표현으로 정리한다.

## 검증

- 문서 구조를 기존 `docs/specs/v3/prds/phase17-19` 형식과 맞춰 작성
- 파일 경로 및 링크가 프로젝트 문서 구조와 일치하도록 반영
