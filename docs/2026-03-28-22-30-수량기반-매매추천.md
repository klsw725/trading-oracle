# 수량 기반 매매 추천 + 분할 매도 + 현금 연동

## 프롬프트

> 매수금액과 매도 금액을 알려줄때 포트폴리오 기준으로 수량도 알려줄수 있나? 전액을 매도 해야 하는 경우도 있겠지만 분할 매도 분할 매수도 있지 않은가?
> 포트폴리오에 예산을 기입할것 아닌가? 그러면 그 예산에 따라서 몇주를 얼마에 사야 하는지 알려달라. 물론 매도도 동일하다

## 변경 사항

### 1. 분할 매도 (`src/portfolio/tracker.py`)
- `remove_position()`에 `shares` 파라미터 추가
- `None`이면 전량 매도 (기존 동작), 숫자면 해당 수량만 매도
- 잔여 수량이 있으면 포지션 유지 (평단가 보존, market_value/pnl 재계산)
- 보유 수량 초과 시 `ValueError`

### 2. CLI 분할 매도 (`main.py`)
- `remove` 명령에 `--shares`/`-n` 옵션 추가
- 사용법: `uv run main.py remove 005930 --price 60000 -n 5` (5주만 매도)
- 분할 매도 시 잔여 수량, 매도 대금, 현금 잔고 표시

### 3. 현금 연동 (`main.py`)
- `cmd_add()`: 매수 대금(`price × shares`)을 cash에서 차감 (부족 시 경고)
- `cmd_remove()`: 매도 대금(`sell_price × shares`)을 cash에 가산

### 4. LLM 수량 추천 강화 (`src/agent/prompts.py`)

**시스템 프롬프트 변경**:
- 4단계를 "종합 전략 (수량 필수)"로 강화
- 매도/매수/분할 각각에 대해 "N주 중 M주 매도 → 매도 대금 X원" 형태의 산술적 계산 지시
- 각 행동 후 예상 현금 잔고 명시 지시

**유저 프롬프트 변경**:
- "💰 매수력 참고" 섹션 추가: 가용 현금 기준으로 각 후보 종목의 최대 매수 가능 수량 사전 계산
- 프롬프트 말미에 수량/금액 산술 계산 필수 지시
