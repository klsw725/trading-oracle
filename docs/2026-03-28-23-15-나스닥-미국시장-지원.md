# 나스닥/미국 시장 지원 추가

## 프롬프트

> 코스피, 코스닥, 말고 나스닥을 하는 방법은 어떻게 하는것인가?
> B안 yfinance 추가

## 설계: B안 — 티커 포맷 자동 판별

한국 종목: 6자리 숫자 (`005930`), 미국 종목: 알파벳 1-5자 (`AAPL`, `MSFT`).
`is_us_ticker()` 하나로 전체 파이프라인 자동 분기. config 변경 불필요.

## 변경 파일

### `pyproject.toml`
- `yfinance>=0.2.40` 의존성 추가

### `src/data/market.py`
- `is_us_ticker(ticker)`: 알파벳 1-5자면 US
- `fetch_ohlcv()`: US → `fdr.DataReader()`, KR → `pykrx`
- `get_ticker_name()`: US → `fdr.StockListing('NASDAQ')` + `NYSE` 캐싱
- `fetch_market_cap()`: US → `yfinance` `info['marketCap']`

### `src/data/fundamentals.py`
- `fetch_us_fundamentals()`: yfinance로 PER/PBR/배당수익률
- `fetch_fundamentals_cached()`: US 티커 자동 분기
- 배당수익률: `trailingAnnualDividendYield` 우선 사용 (yfinance `dividendYield`는 퍼센트 혼동 이슈)

### `src/common.py`
- `collect_market_data(include_us=True)`: 나스닥(IXIC)/S&P500(US500) 지수 추가 수집
- `has_us_tickers()`: 종목 세트에 US 종목 포함 여부 판별
- `analyze_ticker()`: `fetch_naver_fundamentals` → `fetch_fundamentals_cached`로 변경 (US 자동 분기)

### `main.py`
- `cmd_analyze()`: 종목 수집 후 US 여부 판단 → `collect_market_data(include_us=True)`
- 지수 출력에 나스닥/S&P500 추가

### `src/agent/prompts.py`
- 나스닥/S&P500 지수 프롬프트 포함

### `src/screener/leading.py`
- `screen_leading_stocks(market="NASDAQ")` 지원

## 사용법

```bash
# 미국 종목 분석 (자동 감지)
uv run main.py -t AAPL MSFT NVDA

# 한국+미국 혼합
uv run main.py -t 005930 AAPL

# 미국 종목 매수 기록
uv run main.py add AAPL 200 10 --reason "AI 수혜"
```
