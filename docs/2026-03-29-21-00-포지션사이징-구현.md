# 포지션 사이징 — BUY/SELL 매매 전략 출력 구현

## 프롬프트

> buy와 sell시 알려달라. 수량 추천은 무엇을 기준으로? → 손절 기반 + 합의 강도 + 포트폴리오 레벨 체크

## 변경 내용

### 신규 파일
- `src/portfolio/sizer.py`: 포지션 사이징 모듈
  - `check_portfolio_health()`: 포트폴리오 건전성 평가 (총 손실, 현금 비중, 집중도)
  - `compute_buy_plan()`: BUY 분할 매수 전략 (손절 기반 + 합의 강도별 1차 비율)
  - `compute_sell_plan()`: SELL 분할 매도 전략 (합의 강도별 + 손절매 우선)
  - `compute_action_plan()`: 통합 엔트리포인트

### 수정 파일
- `config.yaml`: `position_sizing` 섹션 추가 (risk_per_trade, first_tranche, sell_ratio, cash_floor 등)
- `main.py`:
  - `_print_consensus_card()`: BUY/SELL 시 매매 전략 블록 출력
  - `cmd_analyze()`: sizer 호출하여 consensus에 action_plan 부착, 포트폴리오 경고 출력
- `src/common.py`: `run_recommend()`에서 BUY 추천에 action_plan 부착
- `scripts/recommend.py`: 터미널 출력에 매수 전략 표시

### PRD
- `.sisyphus/plans/position-sizing-prd.md`: 전체 설계 문서

## 핵심 로직

### BUY (분할 매수)
1. 손절 기반 목표 수량: `max_risk(자산×risk_pct) / (현재가-손절가)`
2. 포트폴리오 제약: 종목당 비중 상한, 가용 현금(현금 하한 보호)
3. 합의 강도별 1차 매수: 만장일치 50%, 강한 33%, 약한 25%

### SELL (분할 매도)
1. 손절매 도달 → 전량 즉시 매도 (최우선)
2. 합의 강도별: 만장일치 전량, 강한 2/3, 약한 1/3
3. 포트폴리오 레벨 조정: 총 손실 초과 시 매도 상향, 현금 비중 부족 시 비중 축소

### 포트폴리오 체크
- 총 손실 한도: -10% 초과 시 최악 종목 감축
- 현금 비중 하한: bull 20%, bear 30%, sideways 25%
- 종목 집중도: 종목당 최대 33%
- 최대 보유 종목 수: config.max_positions
