# 나스닥/S&P500 데이터 LLM 전달 누락 수정

## 프롬프트

> 실행시 환경에서 코스피 코스닥이 나오는데 국장의 경우 미장과 연결이 되어 있는데 나스닥 수치나 이런걸 고려하지 않고 계산하는것 같다 맞는가?

## 문제

두 가지 버그로 나스닥/S&P500 데이터가 LLM 프롬프트에 도달하지 않았음:

1. **수집 단계**: `collect_market_data(include_us=False)`일 때 나스닥/S&P500 수집 자체를 안 함. 한국 종목만 분석해도 미국 지수는 매크로 인과 분석에 필수.
2. **전달 단계**: `market_context` 조립 시 `kospi`, `kosdaq`만 복사하고 `nasdaq`, `sp500`을 빠뜨림 (2군데). 수집해도 LLM에 전달 안 됨.

## 변경 사항

### `src/common.py`

- `collect_market_data()`: 나스닥/S&P500을 `include_us` 조건 밖으로 이동 → 항상 수집. `include_us`는 미국 종목 데이터 수집/레짐 감지 폴백에만 영향.
- `run_analysis()` (line ~303): `market_context` 조립에 `nasdaq`, `sp500` 추가
- `run_single_perspective()` (line ~375): 동일하게 수정

### `main.py`

- 시장 환경 출력 메시지: 항상 "코스피/코스닥 + 나스닥/S&P500" 표시 (조건 분기 제거)

## 영향

- `macro.py`의 `_build_user_prompt()`에서 `for key in ("kospi", "kosdaq", "nasdaq", "sp500")` 루프가 이제 실제로 나스닥/S&P500 데이터를 LLM에 전달함
- 한국 종목 분석 시에도 미국 지수 데이터가 매크로 인과 판단에 활용됨
- API 호출 2회 추가 (나스닥 + S&P500 지수 조회) — FDR 호출이라 비용 없음
