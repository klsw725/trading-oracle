> ValueError: cannot convert float NaN to integer  'uv run main.py' 실행시 발생

## 문제

- `fdr.StockListing("KRX")`에서 일부 종목의 `Marcap` 또는 `Stocks`가 `NaN`으로 들어올 수 있음.
- 기존 코드는 이를 그대로 `int(...)`로 변환해서 `ValueError: cannot convert float NaN to integer`로 중단됨.

## 변경 사항

### `src/data/market.py`

- `_safe_int()` 헬퍼 추가
- `fetch_market_cap()`에서 `Marcap`, `Stocks`를 직접 `int(...)` 하지 않고 `_safe_int()`로 변환
- `_fetch_market_cap_us()`도 동일 헬퍼를 사용하도록 맞춤

## 결과

- `NaN`/`None` 값은 `0`으로 처리되어 분석이 계속 진행됨
- 검증 커맨드 `uv run python -c "from src.data.market import fetch_market_cap; print(fetch_market_cap('373220'))"` 실행 결과 크래시 없이 `{'market_cap': 0, 'shares': 234000000}` 반환 확인

## 참고

- `lsp_diagnostics`에는 이 파일의 기존 기반 타입 경고/오류(`dict` 제네릭, 서드파티 stub 부재, `_US_LISTING_CACHE` 재할당 등)가 남아 있음. 이번 수정이 만든 신규 런타임 오류는 아님.
