# PRD: Phase 21 — 추천 포지션 사이징 가시성

> **SPEC 참조**: [../SPEC.md](../SPEC.md)
> **상태**: 📝 초안
> **우선순위**: P1
> **선행 조건**: Phase 20 추천 후보군 확장 + 다양성 선택(완료)

---

## 문제

현재 `scripts/recommend.py`는 추천 결과에 `매수 가능`, `1차 수량`, `투자금`을 보여주지만,
그 숫자가 어떤 포트폴리오 제약으로 계산됐는지는 거의 드러나지 않는다.

이 구조는 다음 문제를 만든다.

1. **설명 단절**
   - 사용자는 현금이 충분해 보이는데 왜 1~2주만 제안되는지 이해하기 어렵다.
   - 실제 계산은 `현재 현금`이 아니라 `현금 하한을 제외한 가용 현금(available_cash)` 기준인데, 출력에서는 보이지 않는다.

2. **추천 신뢰 저하**
   - `매수 가능` 배지가 떠도 “왜 가능한지”가 보이지 않는다.
   - `매수 불가`도 차단 사유만 일부 보일 뿐, 포트폴리오 상태와 연결이 약하다.

3. **사후 검증 어려움**
   - 사용자는 추천 결과만 보고는 `현금 하한`, `총자산`, `종목 비중 제한`, `리스크 제한` 중 어떤 제약이 지배적이었는지 알 수 없다.

## 솔루션

`scripts/recommend.py` 출력에 **포트폴리오 사이징 컨텍스트**와 **종목별 제약 근거**를 함께 노출한다.

```text
🎯 종목 추천
  포트폴리오: 총자산 12,300,000원 / 현재 현금 8,000,000원 / 현금 비중 65.0%
  매수 가능 현금: 4,925,000원 (현금 하한 25% = 3,075,000원 보호)

🟢 삼성전자 (005930)
  매수 전략 ...
    제약 기준: 리스크 12주 / 비중 9주 / 현금 7주
    최종 제한: 가용 현금
```

## 설계 원칙

- **기존 사이징 로직 불변**: 계산식은 바꾸지 않고, 이미 계산한 값을 보여주는 데 집중한다.
- **공통 + 개별 설명 분리**: 포트폴리오 전체 상태는 상단 공통 요약으로, 종목별 제약은 카드 내부에서 설명한다.
- **JSON/터미널 동기화**: 터미널 출력뿐 아니라 `run_recommend()` 반환 결과에도 동일 메타데이터를 담는다.
- **외과적 변경**: 추천 파이프라인/다관점 분석 로직은 유지하고, 반환 데이터와 표시만 확장한다.

## 마일스톤

### M1: 포트폴리오 사이징 메타데이터 노출

**목표**: `run_recommend()` 결과에 추천 실행 시점의 포트폴리오 사이징 상태를 포함한다.

**변경 파일**:
- `src/common.py`
- `src/portfolio/sizer.py`

- [ ] `check_portfolio_health()` 반환값에 `cash_floor_amount` 포함
- [ ] `run_recommend()` 반환값에 `portfolio_sizing` 추가
- [ ] `portfolio_sizing`에는 최소 다음 값 포함:
  - `cash`
  - `cash_display`
  - `cash_ratio`
  - `cash_floor`
  - `cash_floor_amount`
  - `available_cash`
  - `total_assets`
  - `can_buy`
  - `buy_block_reason`

**검증 기준**:
- 추천 결과만 봐도 현재 현금과 실제 가용 현금의 차이를 이해할 수 있다.

---

### M2: 종목별 제약 근거 노출

**목표**: 추천 카드가 “왜 이 수량이 나왔는지”를 설명한다.

**변경 파일**:
- `src/portfolio/sizer.py`
- `scripts/recommend.py`

- [ ] BUY action plan에 제약별 최대 수량 메타데이터 추가
  - `share_limits.risk`
  - `share_limits.weight`
  - `share_limits.cash`
- [ ] 최종 수량을 결정한 제약을 `binding_constraints`로 노출
- [ ] 추천 카드에 `제약 기준` / `최종 제한` 출력 추가

**검증 기준**:
- 사용자가 “1차 수량/목표 수량이 왜 작게 나왔는지”를 카드만 보고 이해할 수 있다.

---

### M3: 문서/CLI 가이드 반영

**목표**: 추천 출력이 포트폴리오 제약을 보여준다는 사실을 문서에 반영한다.

**변경 파일**:
- `README.md`
- `docs/specs/recommend/SPEC.md`

- [ ] 추천 결과 메타데이터 예시에 `portfolio_sizing` 추가
- [ ] 추천 파이프라인 설명에 포트폴리오 사이징 가시성 반영

**검증 기준**:

## 데이터 흐름

```text
load_portfolio()
    │
check_portfolio_health()
    ├─ 현재 현금
    ├─ 현금 하한 금액
    └─ 가용 현금
    │
compute_buy_plan()
    ├─ 리스크 기준 최대 수량
    ├─ 비중 기준 최대 수량
    └─ 현금 기준 최대 수량
    │
run_recommend()
    ├─ portfolio_sizing 메타데이터 포함
    └─ action_plan 제약 근거 포함
    │
scripts/recommend.py
    ├─ 상단 포트폴리오 요약 출력
    └─ 종목별 제약 근거 출력
```

## 리스크

| 리스크 | 영향 | 완화 |
|--------|------|------|
| 출력 정보 과다 | 추천 화면이 복잡해짐 | 공통 요약 2줄 + 카드 내부 1~2줄로 제한 |
| USD 현금/가격 단위 혼동 | 미국 종목에서 숫자 해석 혼란 | 기존 `price_display` 유지 + 현금은 KRW 기준 설명 |
| 제약 동률 | 어떤 제한이 우선인지 애매함 | `binding_constraints`를 복수 배열로 노출 |

## 성공 기준

- 사용자가 `scripts/recommend.py` 출력만 보고 다음을 이해할 수 있다.
  - 현재 총자산/현금/현금 비중
  - 현금 하한 때문에 실제 가용 현금이 얼마인지
  - 종목별 목표 수량이 리스크/비중/현금 중 무엇에 의해 제한됐는지
- 기존 추천 선정 로직과 매수/매도 계산식은 바뀌지 않는다.
