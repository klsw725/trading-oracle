# PRD: Phase 13 — 숙의 합의 시스템 (MAXS-deliberation)

> **SPEC 참조**: [SPEC.md §5 (숙의 합의 시스템)](../SPEC.md#5-숙의-합의-시스템-maxs-deliberation--phase-13)
> **상태**: ✅ 완료 (M1~M5 구현)
> **우선순위**: P2 — Phase 11과 병렬 착수 가능
> **선행 조건**: 없음 (독립 착수 가능. Phase 12 완료 시 시너지.)

---

## 문제

v1의 합의도 시스템은 5개 관점이 독립적으로 1회 판정한 후 단순 투표로 집계한다. 분기(DIVIDED) 시 양측 근거를 나열할 뿐, 관점 간 교차 검증이 없다. 소수 측이 다수 측의 근거를 봤다면 판단이 바뀔 수 있는데, 그 기회를 주지 않는다.

## 솔루션

MAXS 논문의 Lookahead + Convergence 개념을 차용하여, **분기/약한 합의 시에만** 소수 측 관점에 다수 측 근거를 제시하고 재판정하는 숙의 라운드를 추가한다. 수렴하면 조기 종료하여 비용을 절감한다.

---

## 마일스톤

### M1: 숙의 엔진 코어
- [ ] `src/consensus/deliberator.py` 생성
- [ ] 발동 조건: "분기" 또는 "약한 합의" 시에만 (§5-2)
- [ ] 소수 측 관점 식별 + 다수 측 reasoning 수집
- [ ] 소수 측 재판정 LLM 호출 (숙의 프롬프트 §5-5)
- [ ] 결과 파싱: verdict 변경 여부 + 이유

**검증**: 분기 시나리오(BUY 3 / SELL 2)에서 숙의 발동 확인. 소수 측 2개 관점에 재판정 요청 전송.

### M2: 수렴 판정 및 라운드 관리
- [ ] δ 계산: |변경 관점 수| / |전체 유효 관점 수|
- [ ] 수렴 조건: δ ≤ 0.1 → 종료
- [ ] 최대 3 라운드 → 강제 종료 (미수렴 상태 보고)
- [ ] 라운드 히스토리 기록 (각 라운드의 verdict + 변경 사유)

**검증**: Round 1→2에서 변경 0건 시 즉시 종료 확인. Round 3 도달 시 강제 종료 확인.

### M3: scorer.py 연동
- [ ] `scorer.py`의 `compute_consensus()`에서 분기/약한 합의 감지 시 deliberator 호출
- [ ] 숙의 결과로 vote_summary 갱신
- [ ] 숙의 과정을 출력 JSON에 포함 (§5-6 형식)
- [ ] `--no-deliberation` 플래그로 비활성화

**검증**: 분기 시 숙의 발동 → 합의도 변경(또는 유지) → 출력에 deliberation 섹션 포함.

### M4: 비용 제어 및 안전장치
- [ ] 숙의는 소수 측만 재호출 (다수 측 비재호출)
- [ ] 라운드당 최대 LLM 호출 = 소수 측 관점 수 (최대 2)
- [ ] 전체 최대 추가 호출 = 4 (Round 2: 2 + Round 3: 2)
- [ ] 타임아웃: 숙의 전체 30초. 초과 시 Round 1 결과로 폴백.

**검증**: 만장일치/강한 합의 시 숙의 미발동 (추가 비용 $0) 확인. 타임아웃 시 v1 결과 반환 확인.

### M5: 출력 형식 및 문서화
- [ ] 숙의 결과 출력 형식 (§5-6) 구현
- [ ] SKILL.md 갱신 (숙의 관련 사용자 가이드)
- [ ] 합의도 해석 가이드에 "숙의 합의" 추가

**검증**: shacs-bot에서 숙의 결과 JSON 파싱 가능. "숙의 합의" 라벨이 출력에 표시.

## 의존성

- 기존 `src/consensus/voter.py`, `scorer.py`
- LLM 호출: `src/perspectives/base.py`의 `call_llm()`

## 비용

- 합의/강한 합의: 추가 비용 $0
- 분기 시 (숙의 발동): +$0.04~$0.16
- 월 평균 (분기 30% 가정): +$0.72

## 리스크

- **동조 편향(Anchoring)**: 소수 측이 다수 근거를 보고 무비판적으로 동조할 수 있음. 프롬프트에 "유지 시 반론 필수"를 명시하여 완화.
- **확증 편향 증폭**: 숙의가 항상 다수 쪽으로 수렴하면, 소수 의견의 가치가 소멸. 성과 추적(Phase 4)에서 "숙의 전 소수 측이 맞았던 비율"을 모니터링하여 검증.
- **비용 폭주**: 모든 종목에서 분기가 발생하면 비용이 급증. 일일 숙의 발동 상한(최대 3종목)을 설정하여 방지.
