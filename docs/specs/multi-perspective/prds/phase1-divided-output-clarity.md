> **선행 조건**: Phase 1 다관점 투자 판정 시스템 구현 완료

---

## 문제

`uv run main.py`의 다관점 합의 결과에서 `DIVIDED`가 그대로 표시되면 사용자는 이를 “분할 매수/분할 매도” 지시로 오해할 수 있다. 특히 1주만 보유한 종목에서도 `DIVIDED`가 나타나면 “1주를 몇 주로 나눠 팔라는 것인가?”라는 혼란이 생긴다.

실제 시스템 계약상 `DIVIDED`는 매매 실행 지시가 아니라 `BUY / SELL / HOLD` 관점이 갈린 **분기 상태**다. 따라서 `DIVIDED`에는 자동 매매 수량·가격 계획이 없어야 하며, 사용자는 합의가 없다는 사실을 명확히 이해해야 한다.

---

## 목표

1. CLI 출력에서 `DIVIDED`를 “분할”처럼 보이지 않게 한다.
2. `DIVIDED` 상태가 매매 지시가 아니라 판단 갈림이라는 점을 설명한다.
3. 보유 종목, 특히 1주 보유 종목에서는 분할 매도 불가를 명시한다.
4. 기존 `BUY`/`SELL` action plan 계산과 JSON 데이터 계약은 변경하지 않는다.

## 비목표

- 합의도 계산 알고리즘을 바꾸지 않는다.
- `DIVIDED`에 임의의 매수/매도 수량을 계산하지 않는다.
- LLM 프롬프트나 관점별 verdict 생성 로직을 바꾸지 않는다.
- 기존 JSON 소비자를 위해 `consensus_verdict = "DIVIDED"` 값은 유지한다.

---

## 사용자 영향

### 변경 전

```
📊 삼성전자 (005930) — DIVIDED
```

사용자는 `DIVIDED`를 “분할 매도/매수”로 해석할 수 있다.

### 변경 후

```
📊 삼성전자 (005930) — 판단 갈림

🔶 판단 갈림 안내
  매수/매도 합의가 없어 자동 수량·가격 계획을 계산하지 않습니다.
  현재 보유: 1주
  1주 보유 상태에서는 분할 매도 대신 관망 또는 전량 1주 매도만 가능합니다.
```

사용자는 `DIVIDED`가 실행 지시가 아니라 합의 실패 상태임을 알 수 있다.

---

## 요구사항

### R1. 사용자 표시명

터미널 카드 제목과 관점별 verdict 표시에서 `DIVIDED`는 `판단 갈림`으로 표시한다. 내부 데이터와 JSON 출력의 `consensus_verdict` 값은 변경하지 않는다.

### R2. DIVIDED 안내 문구

`consensus_verdict == "DIVIDED"`이고 `action_plan`이 없으면 카드 본문에 다음 의미를 포함한 안내를 출력한다.

- 매수/매도 합의가 없어 자동 수량·가격 계획을 계산하지 않음
- 보유 종목이면 현재 보유 수량
- 보유 수량이 1주 이하이면 분할 매도 불가

### R3. 기존 action plan 보존

`BUY`와 `SELL`의 기존 `action_plan` 출력은 유지한다. 특히 `SELL`에서는 기존처럼 권장 매도가, 매도 수량, 잔여 수량을 표시한다.

## 검증 기준

1. `uv run python -m py_compile main.py scripts/daily.py`가 통과한다.
2. 샘플 `DIVIDED` consensus를 `_print_consensus_card()`에 넣으면 카드 제목이 `판단 갈림`으로 표시된다.
3. 1주 보유 포트폴리오를 전달하면 “분할 매도 대신 관망 또는 전량 1주 매도” 안내가 표시된다.
4. `SELL` consensus의 기존 action plan 출력은 변경되지 않는다.
