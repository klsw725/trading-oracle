# PRD: Phase 16 — 프롬프트 자가 튜닝 (LLM 기반 자기 개선)

> **SPEC 참조**: [SPEC.md §4](../SPEC.md#4-phase-16-프롬프트-자가-튜닝)
> **상태**: ✅ 완료
> **우선순위**: P2 — Phase 14 결과 축적 후 착수
> **선행 조건**: Phase 14 완료 + 스냅샷 60개 이상

---

## 문제

관점 프롬프트는 수동으로 작성되어 변경되지 않는다. 시장 상황이 변해도 프롬프트는 고정이므로, 특정 관점의 적중률이 지속적으로 낮아져도 자동 개선 수단이 없다.

## 솔루션

적중률이 지속 저조한 관점의 오답 사례를 LLM이 분석하여 프롬프트 개선 제안을 생성. 사용자 수동 승인 후 적용.

## 마일스톤

### M1: 오답 분석기
- [ ] 부진 관점 식별 (적중률 40% 미만 + 하락 추세)
- [ ] 최근 10건 오답 사례 수집 (추천 vs 실제 결과)
- [ ] LLM에 "왜 틀렸는지" 분석 요청

### M2: 개선 제안 생성
- [ ] LLM이 프롬프트 개선 제안 + diff 형태 출력
- [ ] 예상 개선 효과 추정

### M3: 승인 + 적용
- [ ] CLI에서 제안 표시 + 사용자 Y/N 승인
- [ ] 이전 프롬프트 백업 (롤백 가능)
- [ ] 적용 후 A/B 비교 (30일 후 개선 여부 확인)

## 안전장치
- 자동 적용 없음 — 항상 수동 승인
- 1회/월 실행 제한
- 적중률 60%+ 관점은 대상 아님
