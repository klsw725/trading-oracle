# PRD: Phase 19 — 포트폴리오 상관 리스크 관리

> **SPEC 참조**: [SPEC.md](../SPEC.md)
> **상태**: ✅ 완료
> **우선순위**: P1
> **선행 조건**: Phase 18 백테스트(완료), Phase 6 시장 레짐(완료)

---

## 문제

현재 시스템은 종목별 독립 판정만 수행하고, 포트폴리오 레벨에서 포지션 간 상관 리스크를 관리하지 않는다.

**백테스트에서 확인된 문제:**
- 삼성전자 + SK하이닉스 동시 보유 = 반도체 단일 베팅 (상관 ρ ≈ 0.8+)
- 3종목이 동시에 하락한 구간이 MDD -12.57%의 주요 원인
- `check_portfolio_health()`는 비중 집중만 체크하고, 상관 집중은 무시

**실운용 리스크:**
- BUY 시그널이 동시에 발생하는 종목은 대개 같은 섹터 (모멘텀 팩터 특성)
- 시총 상위 스크리닝 결과도 반도체/자동차에 편중
- 사용자가 의도하지 않은 섹터 집중 포지션이 형성됨

## 솔루션

포지션 간 상관계수를 계산하여 진입 시 고상관 종목 매수를 차단하고, 포트폴리오 분산도를 건전성 지표에 추가.

---

## 마일스톤

### M1: 상관계수 계산 엔진

**목표**: 종목 간 일간 수익률 상관계수를 계산.

**신규 파일**: `src/portfolio/correlation.py`

- [ ] `compute_correlation_matrix(tickers, days_back=60)` → `pd.DataFrame`
  - 종목별 OHLCV 수집 → 일간 수익률 → 피어슨 상관계수 행렬
  - 60일 롤링 윈도우 (기본값)
  - 데이터 부족 종목은 NaN 처리
- [ ] `get_max_correlation(ticker, portfolio_tickers, days_back=60)` → `float`
  - 신규 매수 후보와 기존 보유 종목 간 최대 상관계수
  - 진입 판정에 사용
- [ ] 상관계수 캐시: 당일 1회 계산 후 재사용

**검증 기준**: 삼성전자-SK하이닉스 상관계수 > 0.7 확인

---

### M2: 섹터 분류

**목표**: 종목을 섹터로 분류하여 섹터 집중도를 계산.

**변경 파일**: `src/portfolio/correlation.py`에 추가

- [ ] `classify_sector(name)` → `str`
  - 종목명 키워드 기반 분류 (forex.py의 SECTOR_FX_CLASS와 유사)
  - 분류: 반도체, 자동차, 금융, 방산, 바이오, 화학, IT, 에너지, 기타
- [ ] `compute_sector_concentration(positions)` → `dict`
  - 섹터별 보유 비중 계산
  - 단일 섹터 50% 초과 시 "concentrated" 경고

**검증 기준**: 삼성전자+SK하이닉스 보유 시 "반도체 섹터 집중" 경고

---

### M3: 포트폴리오 건전성 통합

**목표**: `check_portfolio_health()`에 상관/섹터 리스크 지표 추가.

**변경 파일**: `src/portfolio/sizer.py`

- [ ] `check_portfolio_health()` 반환값에 추가:
  ```python
  {
      # 기존 필드 유지
      ...
      # Phase 19 추가
      "correlation_risk": {
          "max_pair_correlation": 0.85,  # 가장 높은 쌍별 상관
          "high_corr_pairs": [("005930", "000660", 0.85)],  # ρ > 0.7 쌍
          "avg_correlation": 0.65,  # 평균 상관
      },
      "sector_concentration": {
          "sectors": {"반도체": 65.0, "자동차": 35.0},
          "most_concentrated": "반도체",
          "concentration_pct": 65.0,
          "is_concentrated": True,  # 단일 섹터 > 50%
      },
      "diversification_score": 0.35,  # 0(완전집중)~1(완전분산)
  }
  ```
- [ ] 건전성 등급에 상관 리스크 반영:
  - 고상관 쌍 존재 + 섹터 집중 → "caution" 이상
- [ ] `can_buy` 판정에 상관 체크 추가:
  - 신규 매수 후보와 기존 보유 중 ρ > 0.7인 종목이 있으면 매수 차단
  - `buy_block_reason: "기존 보유(SK하이닉스)와 상관 0.85 — 섹터 분산 필요"`

---

### M4: 백테스트 엔진 통합

**목표**: 백테스트에서 상관 리스크 체크를 적용하여 MDD 개선 효과 측정.

**변경 파일**: `src/backtest/engine.py`

- [ ] 매수 판정 시 기존 보유 종목과의 상관계수 체크
  - ρ > threshold → 매수 스킵
  - threshold는 `config.yaml`의 `correlation.max_pair_correlation` (기본 0.7)
- [ ] `--no-corr` 플래그로 상관 체크 비활성화 (A/B 비교용)
- [ ] 백테스트 결과에 "skipped_by_correlation" 카운트 추가

**검증 기준**: 상관 체크 ON/OFF 비교 시 MDD 개선

---

### M5: 출력 포맷 통합

**목표**: 일일 분석 출력에 상관 리스크 경고 표시.

**변경 파일**: `src/output/formatter.py`

- [ ] 포트폴리오 건전성 섹션에 상관 리스크 경고 추가:
  ```
  ⚠️ 상관 리스크
    삼성전자 ↔ SK하이닉스: ρ=0.85 (높음)
    반도체 섹터 집중: 65% (분산 필요)
    분산도: 0.35/1.00
  ```
- [ ] BUY 차단 시 사유에 상관 정보 포함

## 데이터 흐름

```
[M1] correlation.py    ──→ 상관계수 행렬 (60일 롤링)
                              │
[M2] correlation.py    ──→ 섹터 분류 + 집중도
                              │
[M3] sizer.py          ──→ check_portfolio_health() 확장
                              │
[M4] engine.py         ──→ 백테스트 매수 필터
                              │
[M5] formatter.py      ──→ 터미널 경고 출력
```

## config.yaml 추가 섹션

```yaml
correlation:
  max_pair_correlation: 0.7    # 이 이상이면 동시 보유 차단
  window_days: 60              # 상관계수 롤링 윈도우
  sector_concentration_limit: 50  # 단일 섹터 최대 비중 %
```

## 안전장치

| 장치 | 설명 |
|------|------|
| 데이터 부족 폴백 | 60일 미만 OHLCV → 상관 체크 스킵 (매수 허용) |
| 단일 종목 포트폴리오 | 1종목 보유 시 상관 체크 불필요 → 스킵 |
| 비파괴 원칙 | 상관 데이터 없어도 기존 파이프라인 100% 동작 |
| 오버라이드 | `--no-corr` 플래그로 비활성화 가능 |

## 리스크

| 리스크 | 영향 | 완화 |
|--------|------|------|
| 과도한 차단 | 수출주 대부분이 고상관 → 매수 기회 상실 | threshold 0.7은 같은 섹터만 차단, 섹터 간은 허용 |
| 상관 비정상성 | 위기 시 모든 자산 상관 1.0 수렴 | 60일 롤링으로 최근 상황 반영 + 레짐별 조정 가능 |
| 섹터 분류 오류 | 복합 사업 기업(삼성전자=반도체+가전) 분류 모호 | 주요 매출 기준 분류 + config 오버라이드 |

## 구현 순서

```
M1 (상관 계산) → M2 (섹터 분류) → M3 (건전성 통합) → M4 (백테스트) → M5 (출력)
```

M1~M2 순차 의존. M3/M4/M5는 M2 완료 후 병렬 가능.
