# 시그널 엔진 v2 구현

**날짜**: 2026-03-29
**프롬프트**: 미장과 국장이 같은 데이터 기반이여서 그런걸까? 구조적으로 어떻게 보완해야 할지 고민해봐 → 진행

## 분석 과정

### 5가지 구조적 결함 발견
1. 시그널 상관성 0.43~0.47 (6개가 실질적으로 3개)
2. BB 압축이 미장에서 96~97% 항상 활성 → 양쪽 공짜 투표
3. 고정 3% 임계값이 변동성 무시 (Apple std 1.39% vs 삼성 3.11%)
4. 볼륨 데이터 미사용 (고거래량 AAPL BULLISH 63% vs 저거래량 45%)
5. 레짐 필터 미연동

### 실험 결과
- BB 투표 변경 (위치 기반): Samsung/NVDA 개선, AAPL/TSLA 악화 → **불채택**
- 볼륨 시그널 투표: 대부분 기권하여 실질 min_votes 상향 → **불채택**
- 동적 min_votes: 명확한 개선 없음 → **불채택**
- ATR 임계값: v1과 유사 성능, 원칙적으로 올바름 → **채택**
- 레짐 필터: 실제 파이프라인에서 역방향 시그널 억제 → **채택**

### 핵심 교훈
> OHLCV 기반 단순 기술 지표로 미국 대형주(AAPL 등)에서 일관된 엣지를 만드는 것은 현실적으로 어려움. 효율적 시장에서의 alpha는 LLM 관점(외부 지식)과 레짐 인식에서 와야 함.

## 변경 사항

### `src/signals/technical.py`
- `compute_signals()`에 `regime` 파라미터 추가 (기본값 None)
- 모멘텀 임계값: 고정 3% → ATR 기반 동적값 (`max(atr_pct * mult, 0.01)`)
- BB 위치(`bb_position`)와 볼륨 비율(`volume.ratio`)을 정보 필드로 추가 (투표 미참여)
- 레짐 필터: bull장에서 약한 BEARISH → NEUTRAL, bear장에서 약한 BULLISH → NEUTRAL
- `calc_bb_position()` 함수 추가
- 반환값에 `threshold_used` 필드 추가

### `src/common.py`
- `analyze_ticker()`, `analyze_tickers()`에 `regime` 파라미터 전달 지원

### `main.py`, `scripts/daily.py`
- 시장 레짐을 `analyze_tickers()`에 전달하도록 연동

### `src/perspectives/quant_perspective.py`
- `_build_signals_dict()`에 bb_position, volume_ratio 정보 추가

### `config.yaml` 신규 옵션
- `signals.momentum_threshold_atr_mult`: ATR 배수 (기본값 1.0)
